# AI Work Template — Универсальный шаблон AI-файлов

> **Версия:** 2.0
> **Назначение:** Базовый набор файлов для любого проекта

---

## 📦 Что это

Это набор из 10 markdown-файлов которые превращают любую сессию с ИИ
в структурированную рабочую среду — как у GhostBrowser.

---

## 📁 Файлы

| Файл | Назначение |
|------|-----------|
| `onboarding.md` | Ввод в курс дела + АВТОНАСТРОЙКА при первом запуске |
| `task.md` | Пользователь пишет задачу сюда |
| `tasks.md` | ИИ расписывает детали задачи |
| `architecture.md` | Архитектура проекта |
| `rules.md` | Правила работы ИИ |
| `user-responses.md` | Ответы пользователя, контекст |
| `design.md` | Дизайн-система |
| `security.md` | Политика безопасности |
| `workflow.md` | Рабочий процесс (подпланы) |
| `roadmap.md` | Дорожная карта |

---

## 🚀 Как использовать

### 1. Скопируй папку `AI_work` в свой проект

```bash
# Windows
xcopy C:\Users\Пользователь\Desktop\AI_work C:\путь\к\проекту\AI /E /I

# Linux/Mac
cp -r ~/Desktop/AI_work /путь/к/проекту/AI
```

Или просто скопируй папку через проводник.

### 2. Открой проект с ИИ

Напиши первый промт:

```
Привет! Это мой проект [название].
Путь: [путь]
Стек: [язык, фреймворк]

Прочитай все файлы из папки AI/ и выполни АВТОНАСТРОЙКУ
из onboarding.md.
```

### 3. ИИ сделает всё сам

- Просканирует проект
- Заполнит все плейсхолдеры `[PROJECT_NAME]`, `[PROJECT_PATH]` и т.д.
- Опишет архитектуру в architecture.md
- Заполнит task.md
- Сообщит когда готов

### 4. Пиши задачи в `AI/task.md`

```
## [моя просьба]
Добавить кнопку X
```

ИИ прочитает все 10 файлов, разобьёт на подпланы и выполнит.

## 🔑 Ключевые принципы

| Принцип | Описание |
|---------|----------|
| **10 файлов** | ИИ читает все перед каждой задачей |
| **Подпланы** | Большая задача → маленькие шаги |
| **Сборка после каждого** | `dotnet build` или аналог |
| **Вопросы после сборки** | ИИ выводит панель с вопросом |
| **Диагностика каждые 3** | Проверка AI-файлов на актуальность |
| **Не выдумывай** | ИИ обязан гуглить если не знает API |

## 📝 Плейсхолдеры (автозаполнение)

При первом запуске ИИ заменит:

| Плейсхолдер | На что |
|-------------|--------|
| `[PROJECT_NAME]` | Название проекта |
| `[PROJECT_PATH]` | Путь к проекту |
| `[BUILD_COMMAND]` | Команда сборки |
| `[RUN_COMMAND]` | Команда запуска |
| `[LANGUAGE]` | Язык программирования |
| `[FRAMEWORK]` | Фреймворк |
| `[ARCHITECTURE]` | Паттерн архитектуры |
| `[АВТО]` | Автозаполнение при сканировании |

---

## ⚙️ АВТОНАСТРОЙКА

При первом запуске ИИ выполнит:

1. **Сканирование** — glob + list_directory + чтение README
2. **Заполнение** — все плейсхолдеры во всех 10 файлах
3. **Architecture** — описание структуры проекта
4. **Report** — отчёт пользователю
