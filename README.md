# 📋 AI Work — Универсальный шаблон AI-файлов для любого проекта

## Что это?

Это **готовый набор AI-файлов** который можно подключить к ЛЮБОМУ проекту.
Скопировал → попросил ИИ настроить → ИИ заполняет реальной информацией → работаешь.

## Как использовать

### 1. Скопируй эту папку в свой проект
```
mkdir MyProject/AI
xcopy AI_work MyProject/AI /E /I /Y
```

### 2. Скажи ИИ: "Настрой AI-файлы под этот проект"

ИИ сам:
- Просканирует структуру проекта
- Заполнит `architecture.md` — описание компонентов
- Заполнит `design.md` — цвета, стили, UI
- Заполнит `task.md` — статус и задачи
- Заполнит `tasks.md` — баги и TODO
- Заполнит `security.md` — уязвимости и защита
- Заполнит `roadmap.md` — план развития
- Заполнит `ideas.md` — идеи для фич с ID
- Заполнит `onboarding.md` — быстрый ввод в контекст
- Заполнит `rules.md` — правила работы с проектом
- Заполнит `workflow.md` — процесс разработки
- Заполнит `user-responses.md` — ответы пользователя
- Заполнит `changelog.md` — история изменений
- Заполнит `bug-handling.md` — правила обработки багов

### 3. Начни работу
Теперь в каждой сессии ИИ будет читать эти файлы и сразу знать контекст.

## Как работает связь ideas → tasks → task

```
ideas.md (БАНК ИДЕЙ — все возможные фичи с ID)
    ↓ (пользователь выбирает ID)
tasks.md (TODO — запланировано к реализации)
    ↓ (пользователь пишет в task.md "сделай [ID]")
task.md (ТЕКУЩАЯ ЗАДАЧА — что делается сейчас)
    ↓ (выполнено)
changelog.md (ИСТОРИЯ — что и когда сделано)
```

**Пример:**
1. Пользователь пишет в `task.md`: "Сделай `STEALTH-001`"
2. ИИ читает `ideas.md` → находит `STEALTH-001` → понимает что нужно сделать
3. ИИ создаёт подпланы в `tasks.md` → выполняет
4. Обновляет `changelog.md` → история изменений

## Структура файлов

| Файл | Назначение |
|------|-----------|
| `onboarding.md` | Быстрый ввод в курс дела (читать первым) |
| `task.md` | Текущая задача от пользователя + связь с ideas.md |
| `tasks.md` | Детализированный список багов и задач + связь с ideas.md |
| `architecture.md` | Полная архитектура проекта |
| `rules.md` | Правила работы ИИ с проектом |
| `user-responses.md` | Ответы пользователя на вопросы |
| `design.md` | Дизайн-система (UI/UX, цвета, стили) |
| `security.md` | Политика безопасности |
| `workflow.md` | Рабочий процесс (разбиение на подпланы) |
| `roadmap.md` | Дорожная карта проекта |
| `ideas.md` | Банк идей с ID для связи с task.md |
| `changelog.md` | История изменений с кодом ДО/ПОСЛЕ |
| `bug-handling.md` | Правила обработки багов |

## Золотые правила для ИИ

1. **Читай ВСЕ файлы перед началом работы**
2. **Не начинай без понимания контекста**
3. **Разбивай большие задачи на маленькие шаги**
4. **Проверяй после каждого изменения**
5. **Обновляй AI-файлы после каждой задачи**
6. **Делай commit каждые 3 успешных изменения**

- ✅ Мобильные (Flutter, React Native)

- ✅ Библиотеки и фреймворки
